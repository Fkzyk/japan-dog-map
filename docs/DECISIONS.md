## 認証方式

### 決定: Supabase Anonymous Sign-Ins を採用する

日付: 2026-04-19

議論の経緯。

- LocalStorageはiOS SafariのITPで7日後に消える
- Cookieも二重保存してもITPの影響を受ける
- device_idもLocalStorageに依存する以上同じ問題
- 完全解決にはアカウント登録が必要だがUXを損ねたくない
- Supabase Anonymous Sign-Ins なら登録不要で auth.uid が Supabase側に永続化される
- 将来 Google/メール連携で「昇格」できる（linkIdentity）

結論。

- Anonymous Sign-Ins を採用
- Lazy Auth（書き込み時のみ認証発動）で MAU を最小化
- 閲覧だけのユーザーは認証しない

## Captcha の採用

### 決定: Cloudflare Turnstile Managed mode（invisible）を採用する

日付: 2026-04-19

議論の経緯。

- Captcha なしだと Bot が大量の匿名ユーザーを作成可能
- Supabase MAU 50,000 超過で $0.00325/MAU の課金が発生
- 完全無料運用の絶対条件に反する
- Visible Captcha はUXを損ねる
- Cloudflare Turnstile Managed mode（invisible）なら通常ユーザーには見えない

結論。

- Cloudflare Turnstile Managed mode を採用
- 通常ユーザーには何も表示されない
- VPN・Tor・Bot挙動の疑わしいアクセスのみチャレンジ表示（全体の数%想定）
- これは完全無料運用を守るための必要最小限の手段

___

## LocalStorage依存の全廃

### 決定: データ保持用途でのLocalStorageは使わない

日付: 2026-04-19

対象データ。

- お気に入り
- チュートリアル表示状態
- PWAプロンプト表示回数・最終表示日
- 初回注意モーダル表示日

理由。

- iOS Safari の ITP で7日間アクセスがないとLocalStorageが自動削除される
- ユーザーのお気に入りが突然消える体験は致命的
- 他の状態管理も同じ問題を抱える

結論。

- すべて Supabase DB 管理に移行する
- 移行には Anonymous Sign-Ins が前提
- キャッシュ用途（API結果の一時保存）のLocalStorage使用は許可

## ゲーミフィケーションのカウント基準

### 決定: 申請時点で仮バッジ、承認後に確定バッジ

日付: 2026-04-19

選択肢。

- A: 管理者承認後にカウント（信頼性重視、即時反応なし）
- B: 申請時点でカウント（即時反応重視、スパム可能性）
- C: 両方（申請で仮バッジ、承認で確定バッジ）

結論: C を採用。

- 投稿即時に仮バッジで派手に演出してユーザー満足度を上げる
- 管理者承認で確定バッジに昇格
- 却下時はカウント再計算で取消

___

## ゲーミフィケーションの認証要件

### 決定: 匿名ユーザーも全員バッジ獲得可能

日付: 2026-04-19

選択肢。

- A: 匿名ユーザーも全員対象
- B: Google連携済みユーザーのみ（昇格動機）
- C: 匿名も可だが昇格で特別バッジ追加

結論: A を採用。

- 登録なしで使える気軽さを優先
- 全員がゲーミフィケーションの恩恵を受ける
- 将来的に昇格特典を追加する選択肢は残す

___

## ゲーミフィケーションの対象範囲

### 決定: 投稿＋訪問＋口コミの3軸すべて

日付: 2026-04-19

選択肢。

- A: 投稿系のみ（最速で効果が出る）
- B: 投稿＋訪問＋口コミ全部（継続率最大化）
- C: エリア制覇系に注力
- D: SNSシェア重視

結論: B を採用。

- 初期ユーザーの継続率を最大化する
- 3軸それぞれで獲得機会を作り、幅広いユーザーに刺さる設計
- エリア制覇・SNSシェアは Phase 後半で追加検討

## 有料化の方針

### 決定: 有料プラン提供なし、広告収入で運用費をペイ

日付: 2026-04-18

選択肢。

- A: 有料プランで運用費を徴収
- B: 全員に広告表示、広告収入で運用費をペイ
- C: フリーミアム（無料＋有料機能）

結論: B を採用。

- ユーザー課金なしで参入障壁を下げる
- 月間50,000MAUで月$150〜$500の広告収入見込み
- Supabase Pro $25 を十分ペイできる水準
- 独自ドメイン年1,500円だけは必要経費として許容

## インフラ運用

### 決定: GitHub Actions + UptimeRobot で二重冗長化

日付: 2026-04-19

### バックアップ

- GitHub Actions で毎日 pg_dump
- GitHub Actions artifacts に30日分保存
- 完全無料

### 1週間pause対策

- GitHub Actions で6時間ごとに curl ping
- UptimeRobot 5分間隔 ping を併用
- 両方無料

#### B. お気に入りをSupabase化

**概要：** 現在LocalStorage保存のお気に入りをDBに移行し、端末をまたいで保持できるようにする

**現状：**
- `getFavorites()` / `toggleFavorite()` がLocalStorageを使用
- `_favoritesCache` によるキャッシュ最適化は実装済み

**設計上の課題（要決定）：**
- ログイン不要サイトのため「誰のお気に入りか」の識別に Anonymous Sign-Ins が必要
- ただし「スマホ変えたら消える」問題は Anonymous では解決しない（Googleログインが必要）

**選択肢：**

| 案 | 方法 | メリット | デメリット |
|---|---|---|---|
| A | Anonymous Sign-Ins + LocalStorage移行 | ユーザー登録不要 | 端末変更で消える |
| B | Googleログイン任意 | 完全な端末越え | ログイン画面が必要 |
| C | QRコード/コード共有 | ログイン不要で引き継ぎ可 | UX複雑 |

**推奨：** まず案Aで実装し、後でGoogleログインオプションを追加する段階的アプローチ

**実装内容：**
1. Supabase：`spot_favorites` テーブル作成（RLS：自分のデータのみ読み書き）
2. Supabase：Anonymous Sign-Ins を Authentication > Settings で有効化
3. index.html：ページ起動時に `sb.auth.signInAnonymously()` でセッション取得
4. index.html：`getFavorites()` / `toggleFavorite()` をDB読み書きに変更
5. LocalStorageの既存お気に入りをDBにマイグレーション

#### C. 残りの system_flags をフロントに組み込む

**概要：** `allow_reviews`・`allow_helpful` がOFFのとき、対応ボタンを非表示にする

**現状：**
- admin.htmlの設定タブには全フラグが表示・切替可能
- index.htmlには `allow_spot_submission` のみ組み込み済み
- `allow_reviews`・`allow_helpful` はフロントへの組み込み未実装

**実装内容（index.html の `applySystemFlags()` に追記するだけ）：**
```javascript
if (!flags.allow_helpful) {
  document.querySelectorAll('.helpful-btn').forEach(function(el) { el.style.display = 'none'; });
}
if (!flags.allow_reviews) {
  var reviewBtn = document.getElementById('review-submit-btn');
  if (reviewBtn) reviewBtn.style.display = 'none';
}
```

**補足：** フロントにレビュー・helpful機能が実装されてから同時にやると効率的
